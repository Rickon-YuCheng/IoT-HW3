# 變更提案：add-spam-classifier-baseline

## 背景
目前專案尚未具備任何垃圾郵件分類能力，也沒有既有的資料處理或機器學習流程。本次提案旨在建立統一的第一階段方案，一次完成基準 SVM 與邏輯斯迴歸模型，以為後續迭代與比較奠定基礎。

## 目標
- 建立一套從資料擷取、前處理到模型訓練的端到端流程。
- 以公開的 `sms_spam_no_header.csv` 資料集訓練垃圾郵件分類模型，並產出可重現的結果。
- 在同一階段完成線性 SVM 與邏輯斯迴歸模型，並比較兩者的核心評估指標。

## 非目標
- 尚未評估或部署到生產環境。
- 不設定 Phase 2 以後的額外階段，改為在本階段內完成基準模型比較。
- 尚未導入模型監測、資料漂移偵測或自動化模型更新。

## 範圍
- 資料來源：`https://raw.githubusercontent.com/PacktPublishing/Hands-On-Artificial-Intelligence-for-Cybersecurity/refs/heads/master/Chapter03/datasets/sms_spam_no_header.csv`
- 模型技術：同一階段內建立線性 SVM 與邏輯斯迴歸兩種模型，以 TF-IDF 為核心特徵工程流程並比較結果。
- 程式語言與框架：尚未指定，建議以常見資料科學工具（例如 Python + scikit-learn）實作。

## 階段規劃（合併後的單一階段）
- 下載並緩存資料集。
- 進行資料探索與清洗，建立訓練/驗證切分策略。
- 建立可重現的特徵工程流程（例如文字前處理、向量化）。
- 以相同資料流程訓練線性 SVM 與邏輯斯迴歸模型，並輸出 accuracy、precision、recall、F1。
- 彙整模型評估結果，說明差異、限制與後續建議。

## 風險與緩解
- **資料品質或偏差問題**：若資料集不平衡或含有雜訊，需透過加權或重採樣技巧緩解，並於 Phase Baseline 中記錄觀察。
- **模型表現不足**：若兩種模型皆未達預期，需於報告中提出後續改善方向（額外特徵工程或其他演算法）。
- **程式語言與工具尚未定稿**：在設計文件中明確列出假設與替代方案，確保未來可持續擴充。

## 未解問題
- 後續是否導入更多模型、模型監測或部署仍待產品/技術團隊進一步定義。
- 模型最終要整合的系統介面與部署環境仍待釐清。
